OSX Basics

Just turn the thing on

  • In case the machines aren't on ahead of time, the power button is in the back
  • You can log on/off using the Apple logo in the top left hand of your screen

Application Equivalence

  • Finder is like Explorer in Windows or Nautilus (et. al.) in Linux
  • Terminal is like DOS in Windows or the Terminal in Linux
  • Spotlight is like the Start bar in Windows 7 or Gnome Do in Linux
  • Our text editor for the course will be Sublime Text 2

Hotkeys

Cmd here mean the Command key. It may be labeled on your keyboard as just the Apple logo. Or if you are using a keyboard intended for Windows, the Command key is the Windows key.

  • Copy - Cmd + C
  • Paste - Cmd + V
  • New Directory - Cmd + Shift + N
  • Switch Applications - Cmd + Tab
  • Switch Between Application Windows - Cmd + ~
  • Opening Spotlight - Cmd + Space

Using the Terminal

The terminal can be opened by hitting Cmd + Space to open Spotlight and then typing Terminal. The first result will be Terminal.app, so you can hit enter to open the application.

Commands

  • List files - ls
  • Change directories - cd [destination]
  • Show current directory - pwd
  • Copy - cp [source] [destination]
  • Move - mv [source] [destination]
  • Delete - rm [file]
  • Delete directory - rm -r [directory]
  • Make directory - mkdir [name]

Using Git

Download the project
  1. Go to the CCAEJS project on github. Copy the URL. This is the location of my version of the project.
  2. Open the terminal and change directories to the Desktop.
  3. Make a new directory and name it after yourself. We'll store all of our files here to keep them separate from the rest of the users at CCAE.
  4. Download the ccaejs project by using the git clone command.
Update the project

At the beginning of every class, you should update your project to be sure you have the current materials.

  1. Open the terminal and change directories to your project folder.
  2. Do a git pull from the master branch of the original project.
  3. You should see a list of all of the recent changes since the last session.
Fork the project
  1. Sign-up/Log-in to GitHub and go to the ccaejs project.
  2. Click on Fork button in the top right.
  3. Open the terminal and change directories to your project folder. Add your fork as a remote.
  4. Type git remote -v. This shows you all the remotes that you know about. The address we used is the location of your remote, and the name you gave (in this case, shelby), is your nickname for it. Origin is the default name for the original source of the project.
Save changes
  1. Open the terminal and change directories to the project folder.
  2. Do a git status to see what files have changed. Is it only what you were expecting? Typically, I'm only expecting you to edit files within your people folder.
  3. Do a git add for all the files you want to save. You can do multiple files in one command
  4. Do a git commit to package these changes for saving. The -m stands for "message" and it is used to describe the changes you are saving.
  5. Try pulling from origin and your own fork to be sure you aren't missing anything.
  6. Push your commit up to your own remote. Later, I will merge it into the main project where it will go live. Your username and password here are the ones registered with GitHub.
  7. Note: You should not see the "authentication failed" message as is in the photo. I do not have access to save to Shelby's fork, so I'm rejected. You own your own fork, so you should see a success message.

Variables

Types

JavaScript is a loosely typed language. These are the types available to you: 

// Numbers - Integars and floats are both Numbers
var x = 2;
var y = 3.14;

// Strings - Literally whatever you type. Double quotes or single quotes are fine.
var name = "Murphy";
var sentece = 'Hi, my name is Adam and I have 2 cats!';

// Booleans
var alwaysTrue = true;
var neverTrue = false;

// Arrays
var list = [1, 2, 3];
var mixedList = ["This is fine", 2];

// Objects
var instructor = {
  name: "Adam",
  favoriteHobby: "Programming"
};

The var keyword

The var keyword is special and pertains to scope. We'll address it later. For now, rules of thumb:

// The first time you define a variable, use var.
var x = 2;

// Whenever referring to that variable again, don't use var.
x = "I'm a string now"

Note: Variables are dynamic in JavaScript. In the above example, x started as a number and then we redefined it as a String.

Conditionals

Basic syntax

// Basic syntax
if (condition) {
  doSomething();
} else if (otherCondition) {
  doSomethingElse();
} else {
  fallBack();
}

Comparison Operators

// Equality and inequality
if ("hello" == "hi")      // False
if ("hello" != "hi")      // True

if (1 == 1)               // True
if (1 != 1)               // False

if (1 == "1")             // True
if (1 != "1")             // False


// Exact equality and inequality
if (1 === "1")            // False
if (1 !== "1")            // True

if (1 === 1)              // True
if (1 !== 1)              // False

if ("hello" === "hello")  // True
if ("hello" !== "hello")  // False


// Greater and less than
if (2 < 4)                // True
if (2 > 4)                // False

if (2 >= 2)               // True
if (2 <= 2)               // True

Logical Operators

// AND
if (age > 12 && age < 20) {
  alert("You are a teenager.");
}

// OR
if (myFood == "apple" || myFood == "orange") {
  alert("This is maybe a fruit. I should have created a more exhaustive list, perhaps.")
}

// NOT
if (!true) {
  // That's the same as false. We'll never get here.
}

// All together now
if ((this || that) && !(here || there))

Arrays

Arrays are just lists of data. They do not have a type, so a JavaScript array can contain Numbers, Strings, Objects, and other Arrays all side by side. The lists are also dynamic, meaning you can add new elements without redefining the array. 

// Creating an array
var list = [1, 2, "three"];
list[4] = "And now there's a fourth element";

// Getting items out of an array (they are 0 indexed)
alert(list[0]);       // Shows 1
alert(list[3]);       // Shows "And now there's a fourth element"

// Length of an array
alert(list.length);   // Alerts 4

// Adding to an array
list.push("A new item");

// Removing from an array
list.pop();

// Find the index of elements
alert(list.indexOf("three"));   // Returns 2
alert(list.indexOf("nope"));    // Returns -1 because it is not found

Loops

// For loops
for (var i = 0; i < 10; i++)
  alert(i);

// For each loops
var list = ["a", "b", "c"];
foreach (var index in list)
  alert(list[index]);

// While loops
while (someCondition) {
  // Keep on doing this until
}

// Break
while (true) {
  break; // This would have been an infinite loop but broke tosses us out of it
}

// Continue
for (var i = 0; i < 10; i++) {
  if (i == 2) {
    continue; // Do nothing at the value of 2. Instead, go to the next iteration of the loop.
  }
  alert("The number is " + i);
}

Functions

Declared functions

Declared functions are a lot like functions as you'd find in C. A big difference is that you don't define the return type or the types of the parameters because we don't need to. Declared functions are defined at parse time. 

// An exponent declared function
function exponent(number, power) {
  // For brevity, let's ignore negative exponents
  var answer = 1;
  for (var i = 0; i < power; i++) {
    answer *= number;
  }

  return answer;
}

Note: there's already a function that does this for you. It's Math.pow(number, power). Check it out!

Function expressions

Function expressions will produce the same return value as their declared function counterparts. The subtle difference is that function expressions are defined at run time. They look like this: 

// An exponent declared function
var exponent = function(number, power) {
  // For brevity, let's ignore negative exponents
  var answer = 1;
  for (var i = 0; i < power; i++) {
    answer *= number;
  }

  return answer;
}

The implication is that function expressions must be evaluated before the function can be called. The function is now accessible by using the variable called exponent.

Anonymous functions and closures

In the prior example of function expressions, we were actually using an anonymous function. We just happened to assign it to a variable. What if we didn't? 

// Calling an anonymous function
(function(message) {
  alert(message);
})("Hello, World!");
}

We just defined and executed an anonymous function. This example is a bit contrived just to demonstrate the syntax, so let's take a look at how anonymous functions might be useful to us. 

// Using anonymous functions for callbacks
function elaborateRoutine(nextSteps) {
  // Here we can do our long and elaborate task asynchronously

  // And when we're done, we invoke the next instruction
  nextSteps();
}

// Using anonymous functions to bind events
function bind(target, event, action) {
  // If event happens on target, run the action function
}
bind("registrationButton", "click", function() {
  // Let's walk through a pop-up registration wizard...
});
