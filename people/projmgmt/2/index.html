git

  • The notes on git are more fleshed out now. I'm mostly copying them here
  • The pattern is this:
    • Clone the project
    • Pull any updates (Every time)
    • Fork the project
    • Push your changes (Every time)

Clone the project

  1. Go to the CCAEJS project on github. Copy the URL. This is the location of my version of the project.
  2. Open the terminal and change directories to the Desktop.
  3. Make a new directory and name it after yourself. We'll store all of our files here to keep them separate from the rest of the users at CCAE.
  4. Download the ccaejs project by using the git clone command.

Pull any updates (Every time!)

  1. Open the terminal and change directories to your project folder.
  2. Do a git pull from the master branch of the original project.
  3. You should see a list of all of the recent changes since the last session.

Fork the project

  1. Sign-up/Log-in to GitHub and go to the ccaejs project.
  2. Click on Fork button in the top right.
  3. Open the terminal and change directories to your project folder. Add your fork as a remote.
  4. Type git remote -v. This shows you all the remotes that you know about. The address we used is the location of your remote, and the name you gave (in this case, shelby), is your nickname for it. Origin is the default name for the original source of the project.

Push your changes (Every time!)

  1. Open the terminal and change directories to the project folder.
  2. Do a git add for all the files you want to save. You can do multiple files in one command
  3. Do a git commit to package these changes for saving. The -m stands for "message" and it is used to describe the changes you are saving.
  4. Try pulling from origin and your own fork to be sure you aren't missing anything.
  5. Push your commit up to your own remote. Later, I will merge it into the main project where it will go live. Your username and password here are the ones registered with GitHub.
  6. Note: You should not see the "authentication failed" message as is in the photo. I do not have access to save to Shelby's fork, so I'm rejected. You own your own fork, so you should see a success message.

Push your changes (Every time!) cont.

Conditionals

  • Just about identical to every other language you've ever seen.
  • Uses else if, not elsif
  • Has one unusual notion - Exact equality
// Basic syntax
if (condition) {
  doSomething();
} else if (otherCondition) {
  doSomethingElse();
} else {
  fallBack("hello");
}

Equality and inequality

if ("hello" == "hi")      // False
if ("hello" != "hi")      // True

if (1 == 1)               // True
if (1 != 1)               // False

if (1 == "1")             // True
if (1 != "1")             // False

Exact equality and inequality

if (1 === "1")            // False
if (1 !== "1")            // True

if (1 === 1)              // True
if (1 !== 1)              // False

if ("hello" === "hello")  // True
if ("hello" !== "hello")  // False

Exact equality and inequality

if (2 < 4)                // True
if (2 > 4)                // False

if (2 >= 2)               // True
if (2 <= 2)               // True

Arrays

  • Arrays have no type in JavaScript
  • Arrays are of dynamic length
  • Arrays can be added together with basic Math operations

Array syntax

// Creating an array
var list = [1, 2, "three"];
list[4] = "And now there's a fourth element";

// Getting items out of an array (they are 0 indexed)
alert(list[0]);       // Shows 1
alert(list[3]);       // Shows "And now there's a fourth element"

// Length of an array
alert(list.length);   // Alerts 4

// Adding to an array
list.push("A new item");

// Removing from an array
list.pop();

Loops

  • JavaScript uses C-style loops. There are some extras that we'll get to next week

For Loop

for (var i = 0; i < 10; i++) {
  alert(i);
}

While Loop

while (someCondition) {
  // Keep on doing this until
}

Break and continue

// Break
while (true) {
  break; // This would have been an infinite loop but broke tosses us out of it
}

// Continue
for (var i = 0; i < 10; i++) {
  if (i == 2) {
    continue; // Do nothing at the value of 2. Instead, go to the next iteration of the loop.
  }
  alert("The number is " + i);
}
